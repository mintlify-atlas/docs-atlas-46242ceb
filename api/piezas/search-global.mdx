title: "Visual search (global)"
description: "Search for visually similar parts across all machines in a plant by submitting a query image."
---

## `POST /piezas/buscar/global`

Performs an AI-powered visual similarity search across every part in the plant. The submitted image is processed by the vision model to generate an embedding, which is then compared against stored embeddings for all three image slots of each part. Results are ranked by visual similarity.

<Note>
  Rate limit: **30 requests per minute** per IP address.
</Note>

<Tip>
  Use the `x`, `y`, `w`, `h` fields to define a region of interest (ROI) within the query image. This focuses the embedding on a specific area and can significantly improve search accuracy when the part occupies only a portion of the frame.
</Tip>

### Headers

<ParamField header="x-planta" type="string" required>
  Plant identifier that determines which database is searched. Accepted values include `Planta 1`, `Planta 2`, `planta1`, `p1`, `norte`, and their aliases.
</ParamField>

### Request body

Submit as `multipart/form-data`.

<ParamField body="imagen" type="file" required>
  Query image to search against. Accepted formats: `image/jpeg`, `image/png`, `image/webp`, `image/heic`, `image/heif`. Maximum size: **5 MB**.
</ParamField>

<ParamField body="limite" type="integer" default="5">
  Maximum number of results to return. Clamped to the range **1–20** server-side regardless of the value sent.
</ParamField>

<ParamField body="x" type="string">
  Left edge of the region of interest as a fraction of image width (0.0–1.0).
</ParamField>

<ParamField body="y" type="string">
  Top edge of the region of interest as a fraction of image height (0.0–1.0).
</ParamField>

<ParamField body="w" type="string">
  Width of the region of interest as a fraction of image width (0.0–1.0, exclusive of 0).
</ParamField>

<ParamField body="h" type="string">
  Height of the region of interest as a fraction of image height (0.0–1.0, exclusive of 0).
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>
  `true` on success.
</ResponseField>

<ResponseField name="data" type="array" required>
  Ordered list of matching parts, from most to least similar.

  <Expandable title="properties">
    <ResponseField name="clave" type="string">
      Unique part identifier.
    </ResponseField>
    <ResponseField name="nombre" type="string">
      Part display name.
    </ResponseField>
    <ResponseField name="maquina" type="string">
      Name of the machine this part belongs to.
    </ResponseField>
    <ResponseField name="imagen" type="string">
      URL of the primary reference image, or `null` if none.
    </ResponseField>
    <ResponseField name="distancia" type="number">
      L2 distance between the query embedding and the closest of the part's three embeddings. Lower values indicate higher visual similarity. Rounded to 4 decimal places.
    </ResponseField>
    <ResponseField name="nivel" type="string">
      Confidence level derived from `distancia`: `"Alta"` (distance &lt; 0.35), `"Media"` (distance &lt; 0.55), or `"Baja"` (distance ≥ 0.55).
    </ResponseField>
    <ResponseField name="detalles" type="object">
      Additional part metadata.

      <Expandable title="properties">
        <ResponseField name="ubicacion" type="string">
          Physical location of the part.
        </ResponseField>
        <ResponseField name="uso_en" type="string">
          Description of the part's use.
        </ResponseField>
        <ResponseField name="proveedores" type="string">
          Supplier information.
        </ResponseField>
        <ResponseField name="tiene_foto" type="boolean">
          `true` if at least one reference image is stored.
        </ResponseField>
        <ResponseField name="imagenes_extra" type="array">
          Two-element array containing the URLs of `imagen_2` and `imagen_3` (each may be `null`).
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -X POST https://api.example.com/piezas/buscar/global \
  -H "x-planta: Planta 1" \
  -F "imagen=@/path/to/foto.jpg" \
  -F "limite=3" \
  -F "x=0.1" \
  -F "y=0.1" \
  -F "w=0.8" \
  -F "h=0.8"
```

### Error codes

| Code | Reason |
|------|--------|
| 400  | The uploaded image has an unsupported MIME type. |
| 400  | The uploaded image exceeds the 5 MB size limit. |
| 400  | The uploaded image is invalid or corrupted. |
| 500  | AI model error during embedding generation. |
| 500  | The generated search vector has incorrect dimensions. |
| 504  | AI inference exceeded the 90-second timeout. |
